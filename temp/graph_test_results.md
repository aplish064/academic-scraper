# 图谱功能测试结果

**测试日期：** 2026-04-19

## API测试

### 发现的问题

#### 问题1: UNION查询列数不匹配 (已修复)
**错误信息：**
```
Code: 53. DB::Exception: UNION different number of columns in queries. (TYPE_MISMATCH)
```

**原因：**
- OpenAlex表有19列
- Semantic表有24列
- `get_merged_papers_sql()` 函数使用 `SELECT *` 进行 UNION ALL，导致列数不匹配

**解决方案：**
修改 `api_server.py` 的 `get_merged_papers_sql()` 函数，只选择两个表共同拥有的字段：
```python
# 只选择两个表都有的字段
common_fields = [
    'doi', 'rank', 'author_id', 'author', 'uid', 'title', 'journal',
    'citation_count', 'tag', 'state', 'institution_id', 'institution_name',
    'institution_country', 'institution_type', 'publication_date'
]
```

**状态：** ✅ 已修复并提交

#### 问题2: 查询超时 (性能问题)
**错误信息：**
```
Code: 159. DB::Exception: Timeout exceeded: elapsed 60665.443356 ms, maximum: 60000 ms
```

**原因：**
- 数据量过大：OpenAlex表有120,549,507条记录，Semantic表有8,414,757条记录
- 复杂的JOIN和聚合操作在完整数据集上执行超时
- ClickHouse默认查询超时时间为60秒

**影响范围：**
- `/api/graph/authors` - 超时
- `/api/graph/stats` - 超时
- `/api/graph/edges` - 未测试（依赖于authors接口）

**建议解决方案：**
1. **添加时间范围过滤：** 默认使用较近的时间范围（如最近3年）
2. **优化查询性能：**
   - 添加适当的索引
   - 使用 `PREWHERE` 而不是 `WHERE` 进行早期过滤
   - 增加 ClickHouse 查询超时时间
   - 使用采样查询进行快速预览
3. **实现分页：** 对于大量数据，实现分页加载
4. **添加缓存：** 对常用查询结果进行缓存
5. **使用物化视图：** 预先计算合作关系数据

**状态：** ⚠️ 需要优化
